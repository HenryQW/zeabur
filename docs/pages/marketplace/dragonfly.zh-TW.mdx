# Deploy Dragon Service

[Dragonfly](https://www.dragonflydb.io/) 是一種簡單、高性能、低成本的內存數據存儲。Dragonfly 與 Redis API 完全兼容，但沒有 Redis 管理的複雜性。

這篇文章將教你如何通過服務市場在 Zeabur 上部署自己的 Dragonfly 服務。

## 步驟一：建立項目

第一步，我們需要建立一個項目。進入 [項目](https://dash.zeabur.com/projects) 頁面，你會看到一個項目列表。

點擊列表末尾的 "創建項目" 按鈕以創建一個新項目。

![Create Project](/deploy/common/create-project.en-US.png)

## 步驟二：部署 Dragonfly 服務

你可以在服務市場中一鍵部署 Dragonfly 服務。

按下 "cmd + k" 或者點擊 "Deploy New Service" 按鈕，然後選擇 "服務市場"。

![Select Prebuilt](/deploy/common/select-prebuilt-service.en-US.png)

在列表中找到 Dragonfly 並點擊它以開始部署。

![Select Dragonfly](/databases/dragonfly/deploy-dragonfly.en-US.png)

## 步驟三：獲取連接信息

當你部署 Dragonfly 服務後，Zeabur 會自動幫你注入相關連接使用的環境變量到其他的服務中。

- `DRAGONFLY_HOST`
- `DRAGONFLY_PORT`
- `DRAGONFLY_PASSWORD`

有時候我們可以使用`DRAGONFLY_URI` 來取代上面的環境變量，例如：

```bash
redis://:${DRAGONFLY_PASSWORD}@${DRAGONFLY_HOST}:${DRAGONFLY_PORT}
```

我們也可以在控制台的 "Instructions" 板塊找到連接信息

![Get Connection Info](/databases/dragonfly/connection-dragonfly.en-US.png)

## 步驟四：開始使用

你可以在你的應用或者客戶端中使用上面的連接信息來連接 Dragonfly 服務。

因為 Dragonfly 與 Redis API 完全兼容，所以你可以使用任何支持 Redis 的客戶端來連接 Dragonfly 服務。

![Connect Dragonfly](/databases/dragonfly/client-dragonfly.en-US.png)
