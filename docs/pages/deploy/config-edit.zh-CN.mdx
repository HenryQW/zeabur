# 配置文件编辑

在运行服务时，可以通过配置文件来自定义服务的一些参数，Zeabur 提供了一个配置文件编辑器，可以通过编辑器来指定配置文件的存储位置和内容。在服务启动时，会自动挂载配置文件。

这里我们使用 [nginx](https://nginx.com/) 作为前置代理 [Botway CE](https://github.com/botwayorg/ce)，来演示配置文件编辑器编辑 ngnix 自定义配置文件的过程。

## 步骤一：创建项目

在 [Zeabur Dash](https://dash.zeabur.com/) 创建 Project。

## 步骤二：创建服务

在上述 Project 中分别创建 Prebuilt Service ，服务名称分别为 `nginx` 和 `botway`。

![创建 Ngnix 服务](/deploy/config-edit/create-nginx-service.en-US.png)

![创建 Botway CE 服务](/deploy/config-edit/create-botwayce-service.en-US.png)

## 步骤三：编辑 Nginx 服务配置文件

1. 在 Services 一栏中，点击 `nginx` 服务的 `Settings` 栏目，点击下拉按钮找到 `Configs` 部分，点击 `Open Config Editor` 按钮，进入配置文件编辑器。

![打开配置文件编辑器](/deploy/config-edit/open-config-editor.en-US.png)

2. 在打开的窗口中，点击 `Add Config file` 按钮，添加一个配置文件。在弹出的窗口中，在 `Config Path` 输入框输入配置文件的路径和名称，这里我们输入 `/etc/nginx/nginx.conf`，点击 `Save Config` 按钮。

![添加配置文件](/deploy/config-edit/create-new-config.en-US.png)

<Callout>
    上方的 `Config Path` 需要输入配置文件的绝对路径，请根据实际情况配置。
</Callout>

3. 根据需求，编辑配置文件的内容，这里我们添加了一个 `server` 配置，用于反向代理 Botway CE 服务。

```nginx
worker_processes  5;
error_log  stderr;
worker_rlimit_nofile 8192;

events {}

http {
  default_type application/octet-stream;
  log_format   main '$remote_addr - $remote_user [$time_local]  $status '
    '"$request" $body_bytes_sent "$http_referer" '
    '"$http_user_agent" "$http_x_forwarded_for"';
  access_log   /dev/stdout  main;
  sendfile     on;
  tcp_nopush   on;
  server_names_hash_bucket_size 128; # this seems to be required for some vhosts

  server {
    listen 80 default_server;
    server_name _;

    location / {
      add_header X-Demo-By zeabur;
    
      proxy_pass http://botway-ce.zeabur.internal:8080;
    }
  }
}
```
上面的文件内容中，`proxy_pass` 指令中的 `botway-ce.zeabur.internal` 是 Botway CE 服务的服务名，这里我们使用了 [Private Network](/deploy/private-networking) 功能，可以通过固定的主机名称来使用内网访问服务。

在上面的文件里我们添加了自定义的日志格式和日志存储位置，并且自定义了若干 nginx 优化参数

4. 点击 `Save` 按钮，保存配置文件。

![保存配置文件](/deploy/config-edit/save-config.en-US.png)

## 步骤四：重启服务

在上述步骤中，我们已经完成了配置文件的编辑，接下来我们需要重启服务，找到 Instructions 栏目，找到 `Restart` 按钮，点击重启服务，以使配置文件生效。

![重启服务](/deploy/config-edit/restart-service.en-US.png)
