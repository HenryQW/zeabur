# 介紹

您可以通過任何語言發送 HTTP 請求 與我們的 GraphQL API / REST API 進行交互。在使用之前，您需要在 [Dashboard](https://dash.zeabur.com) 中創建一個 API Key。

# 創建 API Key

在 [Dashboard](https://dash.zeabur.com/) 中，點擊右側下拉菜單的 `Settings`，然後點擊 左側邊欄 `Developer` , 即可看到開發者選項。

![Developer Panel](/developer/api-key/developer.en-US.png)

找到 `API Key` 區域，點擊 `Generate New Token` 按鈕即可生成一個 API Key。

![Generate API Key](/developer/api-key/generate-api-key.en-US.png)

點擊 `View` 按鈕即可查看 API Key。

![View API Key](/developer/api-key/view-api-key.en-US.png)

點擊 `Copy` 按鈕即可複製 API Key。

![Copy API Key](/developer/api-key/copy-api-key.en-US.png)

# 使用 API Key

## 驗證

Zeabur API 可以選擇使用 API Key 進行身份驗證。保證您已經生成了有效的 API Key。

所有 API 請求都應在 `Authorization` HTTP 頭部中包含您的 API Key ，如下所示：

```http
Authorization: Bearer ZEABUR_API_KEY
```

# 發出 REST API 請求

您可以使用任何 HTTP 客戶端庫或者在您的應用中來發出請求。下面是示例：

您可以將以下命令黏貼到終端中以運行您的第一個 REST API 請求。確保將 `$ZEABUR_API_KEY` 替換為您的 API Key

```bash
curl https://gateway.zeabur.com/api/v1/users/me \
  -H "Authorization: Bearer $ZEABUR_API_KEY" \
  -H "Content-Type: application/json" \
```

此處查詢當前用戶的信息。您應該收到類似於以下內容的響應：

```json
{
    "_id": "64ffffffffffffffffff",
    "name": "Zeabur Zebra",
    "email": "zebra@zeabur.com",
    "username": "zeabra",
    "language": "en-US",
    "githubID": 5000000,
    "avatarUrl": "https://avatars.githubusercontent.com/u/5000000?v=4",
    "createdAt": "2023-08-31T08:14:08.873Z",
    "bannedAt": null,
    "bannedReason": null,
    "emailPreference": {
        "buildFailed": true,
        "deploymentCrashed": true
    },

    "referralCode": "zeabra",
    "discordID": "7000000000000000000",
    "customerID": "cus_xxxxxxxxxxxxxx"

```

現在您已經成功地使用 API Key 進行了身份驗證，您可以使用 API Key 來訪問其他 REST API 了。

GrapQL API 的訪問方式與上述相同，只是請求方式不同。您可以在 [GraphQL Playground](https://gateway.zeabur.com/graphql) 中嘗試 GraphQL API。
